Description

In the last stage, test whether the Random forest algorithm does not overfit when you increase the number of trees. We suggest you try various values for n_trees and check whether that is true.

Except for the number of trees, everything remains the same as in the previous stage: fit the algorithm on the entire training set and calculate the accuracy score using the test set.

Objectives
  1. Fit your RandomForestClassifier on the whole train set using n_trees ranging from 1 to 600 with a step of 1. This process might take a while, keep calm and wait a bit. For each classifier, calculate the resulting accuracy on the test set.
  2. Print the first 20 of the resulting accuracy values rounded to three digits after the dot (for example, 3.141);
  3. Plot the resulting dependence of accuracy from the number of trees. Do you have the expected behaviour of your model?
